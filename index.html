Event Listener

  • O que são eventos de mouse e teclado em JavaScript?

    Eventos são ações disparadas dentro da janela do navegador anexados a algum elemento da página web — pode ser um único elemento, um conjunto de elementos, o HTML carregado na guia atual, ou toda a janela do navegador. Existem vários tipos diferentes de eventos que podem vir a acontecer, como por exemplo:

    • O usuário clicando ou passando o cursor do mouse sobre um certo elemento.
    • O usuário pressionando uma tecla do teclado.
    • Uma pagina da web terminando de carregar.

    Fonte: Dev Media


    Existe uma infinidade de eventos definidos para uso em JavaScript, cada um deles possui um manipulador de eventos(event handler), que é um bloco de código(por exemplo uma função) que será executado quando o evento for disparado. Quando esse bloco de código é definido para rodar em resposta a um evento que foi disparado, nós dizemos que estamos registrando um ouvinte de eventos(event listeners). Os ouvintes escutam o evento acontecendo, e o manipulador é o codigo que roda em resposta a este determinado evento.

    Fonte: MDN Reference

  • O que é uma função de callback (utilizada no Event Listener)?

    Trata-se de uma função que é executada quando algum evento acontece ou depois que o código chega ao estado desejado. Também conhecida como função de retorno, o Callback cria regras dentro de outras funções para que sejam utilizadas no futuro. Esta função é passada como parâmetro e normalmente age de forma assíncrona, ou seja, não é executado imediatamente. A aplicação continuará rodando enquanto espera o momento certo da sua execução.   • Qual a diferença de um Event Listener de atributos HTML (como onclick)?

    A principal diferença é que com o atributo onclick você consegue registrar apenas uma função como callback do evento. Já com o Event Listener, você pode utilizar a quantidade de funções que forem necessárias e todas serão ativadas após a ocorrência do evento. Se tornando assim mais indicado a utilização deste método em determinadas situações.

  • O que é Event Bubbling em JS?

    Quando um evento é acionado em um elemento que possui elementos pai, os navegadores modernos por padrão não executam exatamente a partir do elemento que foi clicado. Na verdade eles executam duas fases diferentes — a fase de captura e a fase de bubbling.
    Na fase de captura o evento começa no ancestral mais externo, analisando se há um manipulador de eventos registrado nele e executa em caso afirmativo. Em seguida vai descendo por toda a hierarquia DOM realizando este processo até chegar no elemento em questão (target).

  • O que faz a função event.stopPropagation?

    Para que não ocorra o event Bubbling explicado anteriormente e não seja executado o manipulador em mais de um elemento indesejadamente utilizamos o stopPropagation(). O objeto de evento padrão tem uma função disponível chamada stopPropagation(), que quando utilizada no objeto de evento de um manipulador, faz com que o manipulador seja executado, mas o evento não borbulha mais acima na cadeia, portanto, mais nenhum manipulador rodará.
